Install

Install TF.ai on Windows

1

Download the release

Open the GitHub Releases page, download the Windows zip, and extract the whole folder.

2

Run the app

Open tf_ai.exe from the extracted folder. Keep the adjacent files next to it.

3

Open AI Settings

Pick either Local Ollama or API, then press Test.

Developer build

If you are building from source, install Flutter and run:

flutter pub get
flutter run -d windows

Cloud models

How to get and use an API key

Get a key

  1. Choose an OpenAI-compatible provider.
  2. Create an account on the provider website.
  3. Open the provider dashboard and create an API key.
  4. Copy the key once and keep it private.
  5. Add billing or free credits if the provider requires it.

Paste it into TF.ai

  1. Open AI Settings.
  2. Select API.
  3. Paste the provider API base URL.
  4. Paste the API key.
  5. Enter the model name and press Test.
API provider examples
Provider API base URL Where to create the key Model field
OpenAI https://api.openai.com/v1 API keys Copy a chat model id from the OpenAI dashboard or docs.
OpenRouter https://openrouter.ai/api/v1 Keys Use an OpenRouter model id, for example openai/gpt-4.1-mini.
Groq https://api.groq.com/openai/v1 API keys Use a Groq model id from the Groq console.
Together AI https://api.together.xyz/v1 API keys Use a Together model id from the model list.

Keep your key private

Do not publish API keys in screenshots, GitHub commits, issue reports, or character cards. TF.ai stores the key locally and sends it only to the API base URL you configure.

Local models

How to run locally with Ollama

Install Ollama

  1. Download Ollama for Windows from ollama.com.
  2. Install it and open a new PowerShell window.
  3. Pull a model that fits your hardware.
ollama pull huihui_ai/qwen3.5-abliterated:9b-q8_0

Connect TF.ai

  1. Open AI Settings.
  2. Select Local Ollama.
  3. Set PC address to http://localhost:11434.
  4. Set Ollama model to the exact name you pulled.
  5. Press Test, then Save.
Local app examples
Local app TF.ai mode Address Notes
Ollama Local Ollama http://localhost:11434 Recommended local path. Use the exact model name shown by ollama list.
LM Studio API http://localhost:1234/v1 Start the local server in LM Studio and load a chat model.
Ollama OpenAI API API http://localhost:11434/v1 Useful if you want to use Ollama through the OpenAI-compatible API mode.
llama.cpp server API http://localhost:8080/v1 Advanced option if you already run a local OpenAI-compatible server.

If Ollama says the port is already in use

That usually means Ollama is already running. Open TF.ai, press Test, and only run ollama serve manually if the connection fails.
